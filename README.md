# Docs Mentor Platform

## Общее описание

Docs Mentor Platform — это RAG-платформа для работы с технической документацией.
Пользователь создаёт «базы знаний» из документации фреймворков/библиотек, создаёт чаты и общается с AI-ментором на основе этих баз.

**Архитектура проекта:**

- **Блок 1 (завершён):** Django + DRF + JWT — пользователи, API, модели.
- **Блок 2 (завершён):** Celery + парсинг + разбивка на чанки + PGVector.
- **Блок 3 (завершён):** FastAPI + RAG-чат + LLM + стриминг.
- **Блок 3+ (завершён):** **Поддержка нескольких LLM провайдеров, шифрование ключей.**
- ****Блок 4 (завершён):** Фронтенд (HTML/SCSS/JS), Docker-упаковка, полировка.**

## Права доступа (текущая логика)

* Пользователь видит и управляет **только своими** KnowledgeBase, Chat, Message, UserAPIKey, UserProviderSettings.
* API требует JWT-токен в заголовке: `Authorization: Bearer <access_token>` (кроме `/health`).
* FastAPI проверяет права через Django API (`/api/auth/me/`).
* Для тестирования через браузер добавлена `SessionAuthentication` (временное решение).

## Статусы KnowledgeBase

| Статус | Описание                                                                             |
| ------------ | -------------------------------------------------------------------------------------------- |
| `pending`  | Создана, ожидает начала парсинга                                 |
| `parsing`  | Идёт парсинг и обработка документации                       |
| `ready`    | Парсинг завершён, чанки сохранены                               |
| `failed`   | Ошибка при парсинге (текст ошибки в поле `error_message`) |

## Архитектура и инфраструктура

### Запуск сервисов через Docker Compose

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg16
    ports:
      - "5432:5432"
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

### Фоновые задачи (Celery)

При создании KnowledgeBase автоматически запускается Celery-задача:

1. Парсинг документации (MkDocsParser или UniversalParser)
2. Разбивка на чанки (правило: <1500 токенов — 1 чанк, иначе чанки по 800 токенов с overlap 150)
3. Сохранение чанков в PGVector
4. Обновление статуса: `pending` → `parsing` → `ready` / `failed`

### Парсеры

| Парсер        | Для каких сайтов           | Автосбор страниц                           |
| ------------------- | ---------------------------------------- | --------------------------------------------------------- |
| `MkDocsParser`    | FastAPI, MkDocs, Material for MkDocs     | ✅ Да                                                   |
| `UniversalParser` | Любые сайты (запасной) | ❌ Нет (ручной список)                     |
| `SphinxParser`    | PyTorch, Python docs                     | ⏸ Заморожен (требует доработки) |

### Логирование

* Все события записываются в файл `logs/docs_mentor.log`
* Также выводятся в консоль (терминалы Django и Celery)

| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **SSE-стриминг отключён** | Используется обычный POST-запрос (`/chat/{chat_id}/ask`). Стриминг не работает стабильно из-за особенностей браузерной буферизации. |
